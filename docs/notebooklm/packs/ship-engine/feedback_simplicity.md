name: simplicity-is-elegance
description: User demands radical simplicity in design — no overhead, no noise, no overbuilding
type: feedback
---

Simple = elegant. The brilliance is in the simplicity. Do not add overhead or noise.

**Why:** User corrected a design that was accumulating features (four emotional states, dual accents, crosshair reticles, OSB buttons, House assignment steps, designer panels). Each individually reasonable, collectively noise. The user's content creation workflow needs to look effortless on camera — complexity reads as clutter, not sophistication.

**How to apply:** When designing interfaces or features, start from the minimum viable interaction and resist adding. Every element must justify its existence. If something can be removed without breaking the core loop, remove it. Prefer fewer panels, fewer states, fewer buttons. Elegance = restraint + precision.
