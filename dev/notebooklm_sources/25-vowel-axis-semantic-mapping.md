# Vowel Axis Semantic Mapping For TRENCH

Source type: interpretive product note

Status: usable as a control metaphor for vowel-style bodies

Constraint: not validated yet as a global engine law

## Core Split

Yes:

- For the vowel filter family, the second control axis can be treated as a physical-shape metaphor.
- In practice that means a control metaphor like body size, cavity size, throat opening, or vocal tract scale.

No:

- Do not promote that mapping into a universal engine doctrine.
- Do not hard-code the rule that `Q` always means mouth cavity size.

## Why This Split Matters

The pasted workflow text frames `Vocal Ah-Ay-Ee` and `Vocal Oo-Ah` as dedicated vocal filters where:

- sweeping `FilFreq` moves vowel state
- modulating resonance changes the apparent mouth cavity
- the resonance lane animates the talking quality

That is useful as a vowel-family interpretation.

But there is a hard limit:

- this claim currently lives in the pasted workflow text
- it is not yet validated as a primary-source fact inside the project corpus
- the project outreach and Rossum notes already warn against promoting ungrounded Q-axis assumptions into doctrine

## Product Rule

Treat `body size` as a body-specific interpretation of the second axis.

Do not treat `body size` as a universal definition of the second axis across TRENCH.

## Vowel Family Mapping

For vocal-style bodies such as `Vocal Ah-Ay-Ee` and `Vocal Oo-Ah`, the second axis can be labeled and understood as:

- body size
- cavity scale
- mouth
- throat opening
- vocal tract scale

This preserves the perceptual role of the control without pretending the mapping is globally proved.

## Non-Vowel Bodies

For other bodies, the second axis may map to different perceptual roles, for example:

- pressure
- tightness
- bite
- yield
- collapse
- dominance transfer

This is closer to the locked Station doctrine anyway: controls should speak in perceptual roles, not raw generic-Q language.

## Clean TRENCH Interpretation

Use body-specific semantics for the second axis.

Examples:

- `Vocal Ah-Ay-Ee` or `Vocal Oo-Ah`: second axis = body size or cavity scale
- acid bodies: second axis = pressure or bite
- bass bodies: second axis = weight or collapse
- carve or null structures: second axis = dominance transfer or yield
- hostile shelves or scrambled P2K-style machines: second axis = whatever perceptual stress law actually matches the body

## Authoring Implication

In forge authoring, a vocal body should be described with an authored sentence such as:

- Morph moves from Ah to Ee while body size tightens.

That keeps the body description aligned with the existing body-law and motion-law doctrine.

## Plugin Labeling Implication

Do not label the vertical axis `Q` when the body behavior is clearly perceptual and body-specific.

For vocal presets, better labels include:

- Body
- Cavity
- Mouth
- Size

For non-vocal bodies, the label should change to match that body's actual stress law.

## Hard Boundary

Validated conclusion:

- in vowel filters, the second axis is not best understood as generic synth resonance
- it behaves more like a physical-shape control metaphor

Not yet validated:

- that TRENCH's second axis should be interpreted as body size everywhere

Working rule:

- use this as a vowel-body semantic mapping
- do not elevate it into engine-wide law without stronger primary-source grounding
